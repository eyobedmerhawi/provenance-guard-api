\# Provenance Guard - Planning Document



\## Project Overview



Provenance Guard is a backend system that helps determine whether a piece of written content is more likely to be AI-generated or human-written. Instead of making a simple yes-or-no decision, the system combines multiple detection signals to produce a confidence score and a transparency label that communicates the level of certainty to users.



The system accepts a piece of text and a creator ID through a submission endpoint. It analyzes the content using two independent detection signals, combines those signals into a confidence score, generates a transparency label, records the decision in an audit log, and returns the results to the user. If a creator disagrees with the decision, they can submit an appeal, which updates the content's status to "under review" and records the appeal in the audit log.


## Detection Signals



\### Signal 1: LLM-Based Classification (Groq)



The first detection signal uses a large language model through the Groq API to analyze the writing style and determine whether the content appears more likely to be AI-generated or human-written. The model returns a confidence score between 0 and 1 representing how strongly it believes the text was generated by AI.



\*\*Strengths\*\*



\* Understands context and writing quality.

\* Detects semantic patterns common in AI-generated text.



\*\*Limitations\*\*



\* Can misclassify highly polished human writing.

\* May struggle with short pieces of text.



\---



\### Signal 2: Stylometric Heuristics



The second detection signal analyzes measurable writing characteristics such as sentence length variation, vocabulary diversity (type-token ratio), and punctuation usage. AI-generated writing often has more consistent structure, while human writing tends to be more varied.



\*\*Strengths\*\*



\* Fast and inexpensive.

\* Independent from the language model.



\*\*Limitations\*\*



\* Cannot understand meaning or context.

\* May incorrectly classify formal human writing as AI-generated.

## Confidence Scoring



Each detection signal produces a score between 0 and 1 indicating how likely the content is to be AI-generated. The final confidence score is calculated by averaging the LLM score and the stylometric heuristic score.



Confidence ranges:



\* \*\*0.70 – 1.00:\*\* High confidence that the content is AI-generated.

\* \*\*0.40 – 0.69:\*\* Uncertain. The signals disagree or neither provides strong evidence.

\* \*\*0.00 – 0.39:\*\* High confidence that the content is human-written.



Using three confidence ranges allows the system to acknowledge uncertainty instead of forcing every submission into a simple AI or human decision.

\## Transparency Labels



\### High-Confidence AI



> \*\*Likely AI-Generated\*\*

>

> Our system has high confidence that this content was generated with the assistance of artificial intelligence.



\### Uncertain



> \*\*Uncertain Attribution\*\*

>

> Our system could not confidently determine whether this content was written by a human or generated by AI. Additional review may be necessary.



\### High-Confidence Human



> \*\*Likely Human-Written\*\*

>

> Our system has high confidence that this content was written by a human author.



\## Appeals Workflow



If a creator believes their content was incorrectly classified, they can submit an appeal using the content ID returned by the submission endpoint. The appeal must include a short explanation from the creator describing why they believe the decision was incorrect.



When an appeal is submitted, the system will:



\* Update the content status to \*\*under\_review\*\*.

\* Store the creator's reasoning.

\* Record the appeal in the audit log.

\* Return a confirmation message.



\---



\## Anticipated Edge Cases



\### Edge Case 1



A highly polished essay written by a human may appear AI-generated because of its consistent writing style and formal language.



\### Edge Case 2



A heavily edited AI-generated document may appear human-written because it contains irregular sentence structure and personal anecdotes.

\## API Endpoints



| Endpoint     | Purpose                                        |

| ------------ | ---------------------------------------------- |

| POST /submit | Submit text for attribution analysis           |

| POST /appeal | Submit an appeal for a previous classification |

| GET /log     | View the structured audit log                  |

\## Architecture



```text

Client

&#x20;  │

&#x20;  ▼

POST /submit

&#x20;  │

&#x20;  ├──► Groq LLM Signal

&#x20;  │

&#x20;  ├──► Stylometric Signal

&#x20;  │

&#x20;  ├──► Confidence Scoring

&#x20;  │

&#x20;  ├──► Transparency Label

&#x20;  │

&#x20;  ├──► Audit Log

&#x20;  │

&#x20;  ▼

JSON Response



Appeal Flow



POST /appeal

&#x20;     │

&#x20;     ▼

Update Status → under\_review

&#x20;     │

&#x20;     ▼

Audit Log

&#x20;     │

&#x20;     ▼

Confirmation Response

```



The submission endpoint receives user content and sends it through two independent detection signals. Their scores are combined into a confidence score, which determines the transparency label shown to the user. Every decision is recorded in the audit log. If a creator submits an appeal, the content status is updated to \*\*under\_review\*\*, and the appeal is also logged.
