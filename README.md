\# Provenance Guard API



\## Overview



Provenance Guard is a Flask-based backend API that estimates whether submitted text is more likely to be AI-generated or human-written. Rather than relying on a single detector, the system combines multiple independent detection signals into a confidence score and presents the result using plain-language transparency labels. Every submission is logged in a structured audit log, and creators can submit appeals if they disagree with a classification.



\---



\# Features



\* Multi-signal AI attribution

\* Groq LLM detection

\* Stylometric heuristic analysis

\* Combined confidence scoring

\* Plain-language transparency labels

\* Structured audit log

\* Appeals workflow

\* Rate limiting

\# Detection Signals



\## Signal 1 — Groq LLM



The first detection signal uses a Groq-hosted large language model to determine whether the writing appears more likely to be AI-generated or human-written.



\### What it captures



\* Writing style

\* Overall coherence

\* Language patterns

\* Semantic structure



\### Limitations



The language model may incorrectly classify highly polished human writing as AI-generated and may struggle with very short pieces of text.



\---



\## Signal 2 — Stylometric Heuristics



The second signal analyzes measurable writing characteristics.



Metrics include:



\* Vocabulary diversity

\* Sentence-length variation

\* Punctuation density



\### What it captures



Stylometric analysis measures consistency in writing style rather than meaning.



\### Limitations



Formal human writing may resemble AI-generated writing, while heavily edited AI text may appear more human.



\---



\# Confidence Scoring



The final confidence score is calculated by averaging:



\* Groq LLM score

\* Stylometric heuristic score



Confidence ranges:



|     Score | Result       |

| --------: | ------------ |

| 0.70–1.00 | Likely AI    |

| 0.40–0.69 | Uncertain    |

| 0.00–0.39 | Likely Human |



\## Validation



To validate the scoring system, multiple writing samples were submitted through the API. Longer, structured AI-style passages consistently produced higher confidence scores, while shorter or more conversational submissions produced noticeably lower confidence values. These tests confirmed that the confidence score changed meaningfully across different writing styles instead of remaining constant.

\# Transparency Labels



\## High Confidence AI



> \*\*Likely AI-Generated:\*\* Our system has high confidence that this content was generated with the assistance of artificial intelligence.



\---



\## Uncertain



> \*\*Uncertain Attribution:\*\* Our system could not confidently determine whether this content was written by a human or generated by AI. Additional review may be necessary.



\---



\## High Confidence Human



> \*\*Likely Human-Written:\*\* Our system has high confidence that this content was written by a human author.

\# API Endpoints



\## POST /submit



Submits text for attribution analysis.



\### Request



```json

{

&#x20; "text": "Example text",

&#x20; "creator\_id": "eyobed"

}

```



\### Example Response



```json

{

&#x20; "attribution": "likely\_ai",

&#x20; "confidence": 0.77,

&#x20; "signals": {

&#x20;   "groq\_result": "AI",

&#x20;   "llm\_score": 0.85,

&#x20;   "stylometric\_score": 0.70

&#x20; },

&#x20; "label": "Likely AI-Generated: Our system has high confidence that this content was generated with the assistance of artificial intelligence.",

&#x20; "status": "classified"

}

```



\---



\## POST /appeal



Allows a creator to appeal a previous attribution decision.



Example request:



```json

{

&#x20; "content\_id": "example-id",

&#x20; "creator\_reasoning": "I wrote this content myself and would like the decision reviewed."

}

```



The content status is updated to \*\*under\_review\*\*, and the appeal is recorded in the audit log.



\---



\## GET /log



Returns the structured audit log containing all submissions and appeals.



\---



\# Rate Limiting



The `/submit` endpoint is protected using Flask-Limiter.



Limits:



\* \*\*10 requests per minute\*\*

\* \*\*100 requests per day\*\*



These limits simulate a realistic writing platform by preventing abuse while still allowing normal users to submit multiple drafts for review.



When the limit is exceeded, the API returns:



```text

429 Too Many Requests

```

\# Audit Log



Every submission records:



\* Timestamp

\* Creator ID

\* Attribution result

\* Confidence score

\* Individual detection signal scores

\* Transparency label

\* Classification status



Appeals are also logged and marked with the status:



```text

under\_review

```

\# Known Limitations



This system may incorrectly classify highly polished human writing as AI-generated because the stylometric signal favors consistent sentence structure and vocabulary usage. Likewise, heavily edited AI-generated content may appear human-written after enough manual revision. Since the system combines only two detection signals, it should be considered an assistive tool rather than a definitive authorship detector.
